These files (`analyze_bottlenecks.py`, `solve_schedule.py`, and `load_neo4j.py`) are early standalone prototypes from an initial Neo4j-first design spike. They are not part of the current product: they are not imported or called by any active module, and `load_neo4j.py` in particular is not runnable as-is.

They are retained here for reference only. Their input file paths are neutral placeholders (e.g. `path/to/sections.xlsx`), so point them at real exports before expecting them to run.
